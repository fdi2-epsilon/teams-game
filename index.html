Quiz collections with "powerful" templating library

Templates are a way to separate quiz content from its view representation: A document.xml just like this is parsed by a template processor and transformed in awesome HTML pages, ready to be displayed to the user. The best part is that you can also define your own template processors using the template API in Java, package them together with your quizzes and use them at runtime.

Navigating the source

The Enigma source code is composed of multiple parts:

  • a common core

    teams-game/common/
    Code that we don't have to write again if we want to support a new platform, but just use it as a library.

  • platform specific view code

    teams-game/android-app/
    The real Android application that can be run on any Android 4.4 device or newer.
    Start looking at MainActivity or EnigmaApplication to get started.

  • game levels

    teams-game/levels/
    Game levels are template directives or HTML content, ready to be packaged by our own tool, equests-gradle, in eqc files ready to be loaded in the app.

Compile n' run

Gradle is an highly customizable build tool and we use it to ease our development cycle.

By simply running ./gradlew installDebug in the root project directory, dependencies are downloaded and the app is compiled and uploaded to a connected Android device or emulator.

If you open the app now, it will show nothing, this is because you need to have some quizzes: if you use the emulator, you can package and upload game levels by using ./gradlew uploadLevels, otherwise you will need to only do ./gradlew collectLevels and manually copy generated collections from android-app/build/adb-push/ to Documents/Enigma/Collections on your phone (create the directories if they do not exist).
